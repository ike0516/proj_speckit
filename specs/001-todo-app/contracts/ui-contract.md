# UI Contract: Simple Todo App

**Branch**: `001-todo-app` | **Date**: 2026-05-11 | **Phase**: 1 — Design

## Overview

本アプリはフロントエンド専用（サーバーサイド API なし）のため、
インターフェースコントラクトは **UI インタラクションモデル** として定義する。

---

## Screen Layout

```
┌─────────────────────────────────────────────┐
│              Simple Todo App                │
├─────────────────────────────────────────────┤
│  [タスクを入力...              ] [追加]       │
├─────────────────────────────────────────────┤
│  ☐  買い物リストを作る           [削除]      │
│  ☑  プロジェクト計画書を書く     [削除]      │
│  ☐  メールを返信する             [削除]      │
├─────────────────────────────────────────────┤
│  (タスクが0件の場合: "タスクがありません")   │
└─────────────────────────────────────────────┘
```

## UI Components

### 1. タスク入力フォーム

| Element       | Type     | Behavior                                                  |
|---------------|----------|-----------------------------------------------------------|
| テキスト入力  | `input[type=text]` | タスク名を入力。placeholder: "タスクを入力..."    |
| 追加ボタン    | `button` | クリックまたは Enter キーで送信                           |

**Validation Feedback**:
- 空または空白のみの場合: 入力欄をハイライト表示し、エラーメッセージを表示
- 送信成功後: 入力欄をクリアし、フォーカスを戻す

### 2. タスクリスト

| Element         | Type       | Behavior                                                 |
|-----------------|------------|----------------------------------------------------------|
| タスク行        | `li`       | 各タスクを1行で表示                                      |
| チェックボックス | `input[type=checkbox]` | クリックで完了・未完了をトグル              |
| タスク名ラベル  | `label`    | チェック済み時: 打ち消し線 + グレーアウト               |
| 削除ボタン      | `button`   | クリックでタスクをリストから即座に除去                   |

**Empty State**: タスクが0件のとき「タスクがありません」のメッセージを表示する

### 3. 完了状態の視覚表現

| State   | Visual                                         |
|---------|------------------------------------------------|
| 未完了  | チェックなし、通常テキスト                     |
| 完了済み | チェックあり、打ち消し線 (`text-decoration: line-through`)、グレーテキスト |

## User Interaction Contract

### タスク追加

```
入力: title (string, non-empty after trim)
前提: テキスト入力欄に1文字以上入力されている
操作: 「追加」ボタンクリック or Enter キー押下
結果:
  - リスト末尾に新しいタスクが表示される (completed: false)
  - 入力欄がクリアされる
  - IndexedDB に永続化される
エラー時:
  - 入力欄に視覚的フィードバック（枠を赤くするなど）
  - エラーメッセージ: "タスク名を入力してください"
```

### 完了状態トグル

```
入力: task.id
操作: チェックボックスのクリック
結果:
  - completed の値が反転する (false → true / true → false)
  - タスク名の表示スタイルが即座に更新される
  - IndexedDB に変更が永続化される
```

### タスク削除

```
入力: task.id
操作: 「削除」ボタンクリック
結果:
  - タスク行がリストから即座に消える
  - IndexedDB から該当レコードが削除される
  - リストが0件になった場合、空状態メッセージを表示する
```
